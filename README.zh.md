# Galaxy View（星系视图）

Obsidian 的电影感 3D 图谱插件——像 NASA [Eyes on Asteroids](https://eyes.nasa.gov/apps/asteroids/#/home) 一样飞行穿越你的库。

你的笔记变成一座星系：按图谱分组着色的辉光节点、纤细低饱和的链接丝线、缓慢旋转的星空背景，以及滑行而非跳切的镜头。对 PKM 而言，颜值与可玩性本身就是生产力的一部分。

> English docs: [README.md](README.md)。

## 亮点

- **真实库上够快**——3,000+ 笔记 / 20,000+ 链接以 60fps、10 个以内 draw call 渲染。力学模拟跑在 Web Worker：布局期间主线程零卡顿。
- **电影感镜头**——点节点飞入（相机偏轴到达并立即环绕，优先扫过邻居密集的一侧）、闲置自动环绕带有机漂移、开场从中心绽放整座星系的创世动画。
- **可玩的力学**——斥力、链接距离/强度、中心引力，以及把点云压成致密亮核 + 旋臂盘的 `coreGravity`/`spiral` 力。拖动滑杆，实时看宇宙重排。
- **曲线连线与深空背景**——连线可弯成远离星系核的轨道弧（滑杆 0=直线）；深空背景拆成四层可自由叠加：星点天幕、星云天幕（一次烘焙、跟配色主题染色）、空间浮星（近大远小的视差层）、集群云雾（贴着稠密星团的彩色云团）。全部融入预设，也可逐层自定义。
- **风格与配色预设**——八套手绘 icon 预设（银河、旋臂、轨道、深空场、星云、极简、烟火、超新星），在背景层/配色/大小/力学/辉光/连线曲率等维度上各有差异、气质分明；悬停实时预览、点击应用。可存自己的预设（排序/删除），另有多套配色主题（哈勃深空、霓虹、落日、赛博、黑客、极光），或从核心 2D 图谱一键导入你的颜色分组。
- **导览**——**漫游**：一键氛围自动巡游，镜头在你重要与久未重温的笔记间穿行；**连接两篇**：选任意两篇笔记，沿最短链接路径飞一遍。
- **聚焦模式**——选中一篇笔记会把其余节点调暗、只留其邻域（可选延伸到二度），满饱和高亮其链接，并弹出可拖动/收起的卡片（含反链、标签、摘要）。
- **Markdown 与 Canvas 同图**——`.md` 笔记和 `.canvas` 画布都进入库图谱；Canvas 保留图谱导航与文件操作，但不会把画布 JSON 伪装成笔记正文。
- **Tag Lens、标签配色与有界枢纽**——标签默认关闭。开启独立的「标签」分区后，可从前 12 个 chip 聚焦图谱、按主标签给笔记着色，或只加入最常用的 5–50 个标签枢纽；三层状态分别持久化，总开关关闭时不做标签计算。
- **六语界面**——English / 中文 / Deutsch / Italiano / Español / Português，自动跟随 Obsidian 语言，面板内可切换。
- **用点的，不用打字**——「过滤」分区列出你的顶层文件夹，带上它们在图里的节点颜色和笔记数：点一下隐藏，悬停出「只看」。它同时是这个图一直缺的配色图例。图例表达不了的横切模式（如 `-file:Index`）交给折叠起来的文本框。
- **搜索与导航**——模糊搜索飞向任意笔记；`WASD`/`Q`/`E` 自由飞行；右键拖或 ⌘ 拖平移；`F`/`R`/`ESC` 快捷键。
- **移动端自适应**——手机上自动降档（无后处理、按度数取前 1500 节点、底部弹层卡片），在 iOS/Android 上保持流畅。
- **两种视觉方向**——恒暗深空，或随主题自适应（含一套设计过的墨纸浅色模式）。

## 安装

**从社区插件商店**：搜索 "Galaxy view"。

**通过 BRAT**：在 BRAT 插件里添加 `Longwind1984/galaxy-view`。

**手动**：从最新 release 下载 `main.js`、`manifest.json`、`styles.css` 到 `<你的库>/.obsidian/plugins/galaxy-view/`，然后在「设置 → 第三方插件」里启用。

## 使用

点左侧 ribbon 的轨道图标，或运行命令 "Open galaxy view"。左上角控制面板依次容纳快捷操作与预设、彼此独立的**「过滤」**和**「标签」**分区、可折叠的「外观 / 深空背景 / 辉光 / 力学」控制、导航与动效区（自动环绕、漫游、连接两篇）以及高级选项。悬停任意滑杆并双击可重置；一切自动持久化。

### 过滤

「过滤」分区列出每个顶层文件夹，带上它的笔记在图里的颜色和数量：

- **点文件夹**＝隐藏它，它的笔记连同相关连线一起离开图。
- **悬停 →「只看」**＝只留这一个文件夹；再点还原。
- **「全部显示」**（分区标题右侧）一键还原。
- 未解析／孤儿两个开关也在这里——它们回答的是同一个问题：哪些笔记进图。

使用文件夹配色时，颜色与图里完全一致（含从 2D 图谱导入的配色组），所以这个分区同时就是配色图例。若开启「标签着色」，图谱会有意切换为主标签颜色；文件夹色点仍作为稳定的文件夹身份，继续服务过滤。

**「＋ 按名字过滤」**（折叠在文件夹列表下面）负责文件夹列表表达不了的、横切各文件夹的模式：

### 过滤语法

| 查询 | 留下 |
| --- | --- |
| `Index` | **路径或文件名**含 "index" 的笔记 |
| `file:Index` | **文件名**含 "index" 的笔记 |
| `path:Daily/` | 路径含 "daily/" 的笔记 |
| `-file:Index` | 文件名含 "index" 之外的**全部** |
| `"star wars"` | 带空格的短语 |
| `Index -path:Daily` | 两个条件同时满足（多词是 AND） |

匹配为大小写不敏感的子串包含。**不搜正文**——过滤器只读路径与文件名，这正是它在大库上能即时响应的原因。

### 标签

标签是独立的探索层，不是另一种文本过滤。打开「显示标签」后读取 Obsidian 既有标签元数据，并显示按笔记数排序的前 12 个标签 chip；已激活标签若后来跌出榜单，会追加保留到被清除为止。单选一个标签即启用 Tag Lens，只让匹配笔记及相关连线保亮；再次点击同一标签则清除。普通节点选择会临时覆盖 Lens，取消选择后自动恢复。

- **标签着色**：按每篇笔记的第一个标签着色。颜色由标签名稳定推导，标签排名变化不会让旧标签换色；无标签笔记保留原文件夹／导入配色。
- **标签枢纽**：只把最常用标签加入图中，并连接到对应笔记。数量滑杆硬限制在 **5–50**；默认关闭，关闭时没有隐藏节点或力学成本。
- **重置标签**：只清除 Lens，并恢复标签着色、枢纽开关和数量默认值，不动过滤、预设或导航。

三项能力复用「显示标签」的同一次 metadata 读取，不增加第二条标签提取链；命名使用「标签枢纽」，避免和深空背景的「星云」层撞名。

### 文件支持

Markdown 笔记沿用 Obsidian 的已解析链接索引。Obsidian 1.13.1 不为 Canvas 暴露同等元数据，因此 Galaxy View 启动时只读一遍 Canvas JSON，把其中的 `file` 卡片解释为「画布 → 库文件」链接；之后仅对发生变化的画布做防抖增量重读，遇到写入中的不完整 JSON 时保留上一次有效结果。标签、摘要等 Markdown 专属元数据不会向 Canvas 请求；Canvas 卡片因此保留日期、图谱统计、打开与聚焦操作，但不会把原始 JSON 当作正文摘要显示。

## 隐私

无任何网络请求、无遥测。插件读取你库的链接图谱，以及（可选、只读）核心图谱在 `.obsidian/graph.json` 里的颜色分组。

## 开发

```bash
npm install
npm run dev      # watch 构建，输出到 ./dev-vault/.obsidian/plugins/galaxy-view/（配合 Hot Reload 自动重载）
npm run build    # 产物在 ./dist/（main.js + manifest.json + styles.css）
npm run lint     # eslint（含 obsidianmd 商店合规规则）
npm run test     # vitest
```

不要直接在 iCloud 真实库里开发——用隔离的本地 dev vault（`./dev-vault/`，已 gitignore；可按需放入 `.md` / `.canvas` 测试夹具）。设计文档见 `docs/design/`，进展见 [WORKLOG.md](WORKLOG.md)。
