# Galaxy View 发布指南

> 给「三个月后没有上下文的我」：怎么把这个本地插件发到 GitHub、给 BRAT 用、最终上架 Obsidian 社区商店。
> 当前状态：代码全部完成并验证通过（M0–M5），只差「对外发布」这一步——这是有意留给你手动做的。

## 0. 一次性前置

GitHub CLI 还没装。装并登录（或全程用网页操作也行）：

```bash
brew install gh
gh auth login        # 选 GitHub.com → HTTPS → 浏览器登录
```

## 1. 把 `<github-user>` 占位符换成你的用户名

`README.md` 安装章节里有一处 `<github-user>/galaxy-view`（BRAT 安装路径）。改成你的真实 GitHub 用户名后再发布。

## 2. 建仓库并推送

仓库**根目录**就是 `/Users/rick/Claude_Code/Galaxy_View`（已是 git 仓库，所有提交就绪）。

```bash
cd /Users/rick/Claude_Code/Galaxy_View
gh repo create galaxy-view --public --source=. --remote=origin --push
```

或手动：在 GitHub 网页建空仓库 `galaxy-view`（public），然后：

```bash
git remote add origin https://github.com/<你的用户名>/galaxy-view.git
git push -u origin main
```

> 注意 `.gitignore` 已排除 `dev-vault/`（你笔记的本地克隆）、`node_modules/`、`main.js`、`dist/`——不会把你的笔记或构建产物推上去。

## 3. 打 tag 触发自动发布

CI（`.github/workflows/release.yml`）监听 tag push：自动跑 lint + test + build，然后把 `main.js`/`manifest.json`/`styles.css` 作为 release 资产发布。

```bash
git tag 0.1.0      # 必须与 manifest.json 的 version 一致，且不带 v 前缀（Obsidian 规范）
git push origin 0.1.0
```

去仓库 Actions 页看 Release 工作流跑完（约 1-2 分钟），Releases 页应出现 `0.1.0` 带三个附件。

## 4. 用 BRAT 安装（beta 分发 / 浸泡测试）

1. 在任意 vault 装 BRAT 插件（社区商店搜 "BRAT"）
2. BRAT → Add beta plugin → 填 `<你的用户名>/galaxy-view`
3. 自动拉取最新 release 并安装

真实 vault 现在装的是手动拷贝的商店版构建；改用 BRAT 后能跟着 release 自动更新，适合发布前再日用一周（浸泡）确认零崩溃。

## 5. 上架社区商店（可选，浸泡满意后）

官方流程（2026 年起走 portal，非旧的 PR-to-obsidian-releases）：

1. 浏览器开 `community.obsidian.md`，用 Obsidian 账号登录，关联 GitHub 证明仓库归属
2. Plugins → New plugin → 同意开发者政策
3. 自动审核给反馈；要改就发新 release（version 必须 bump，`npm version patch` 会自动改 manifest+versions 并打 tag）
4. **提交时先查重**：商店里搜 `galaxy-view` / `Galaxy View` 确认 id 与名称未被占用（id 规则：不得含 "obsidian"，已满足）

商店素材还需要：截图 / GIF（README 里目前没有；找几张深空 + 银河预设 + 聚焦卡片的效果图）。

## 版本号怎么涨

```bash
npm version patch    # 0.1.0 → 0.1.1：自动改 manifest.json + versions.json 并 git tag
git push && git push --tags
```
`versions.json` 记录每个插件版本要求的最低 Obsidian 版本（当前 `0.1.0: 1.7.2`）。
